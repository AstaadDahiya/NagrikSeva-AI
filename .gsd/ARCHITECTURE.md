## Overview

NagrikSeva AI is an AI-powered inbound and outbound phone calling agent for Indian citizen grievance redressal. Citizens call a Twilio number, an AI agent (Gemini 1.5 Flash) converses in Hindi/English/Hinglish, collects complaint details, issues a ticket, and sends WhatsApp confirmation. Administrators monitor everything via a Streamlit dashboard.

## Data Flow

### Inbound Call Flow
1. Citizen dials Twilio number → Twilio `POST /call/incoming`
2. `main.py` initializes Gemini session (`agent.initialize_session`)
3. Returns TwiML greeting with `<Gather input="speech">`
4. Citizen speaks → Twilio STT → `POST /call/respond` with `SpeechResult`
5. `main.py` cleans transcript (`stt.clean_transcript`) → sends to Gemini (`agent.get_response`)
6. Returns TwiML `<Say>` with AI response + `<Gather>` for next utterance
7. Steps 4–6 repeat 4–5 times collecting: name, ward, district, category, description
8. When Gemini includes `NS-XXXXXX-XXXX` ticket ID in response:
   - Extracts structured data from chat history via follow-up Gemini prompt
   - Saves to MongoDB (`database.save_grievance`)
   - Sends WhatsApp confirmation (`whatsapp.send_confirmation`)
9. Call ends → `POST /call/status` → updates call duration, ends session

### Outbound Campaign Flow
1. Admin clicks "Launch Campaign" in Streamlit dashboard
2. Dashboard `POST /outbound/trigger?hours=48` to FastAPI
3. `main.py` fetches old open grievances (`database.get_open_old_grievances`)
4. `outbound.run_campaign` iterates, calling each citizen (1/sec rate limit)
5. Each call plays TwiML from `GET /call/outbound-twiml/{ticket_id}`
6. Outcomes logged to MongoDB (`database.log_outbound_call`)

### WhatsApp Lookup Flow
1. Citizen sends ticket ID to WhatsApp sandbox number
2. Twilio `POST /whatsapp/incoming`
3. `main.py` regex-matches `NS-` pattern → looks up in MongoDB → replies via Twilio
